## 10. 系统设置

系统设置面向软件本身的运行配置，与具体的数据库连接或 Project 无关。

### 10.1 数据存储位置

软件的所有配置数据（数据库连接、Schema 扫描结果、字段生成器配置、Project 定义、生成历史记录等）默认以 SQLite 文件的形式存储在本地。用户需要能够查看当前存储文件的路径，并在需要时将其迁移至其他位置（例如团队共享目录）或切换为外部数据库（MySQL、PostgreSQL、Oracle、MSSQL 及协议兼容的数据库）。

更换存储位置时，系统应支持将现有数据迁移至新位置，以避免已有配置丢失。

### 10.2 LLM 配置

软件中部分能力依赖大语言模型（LLM），例如后续版本规划中的 AI 辅助数据生成（如电影名称、商品描述等自然语言内容的生成）。用户需要能够配置所使用的 LLM 服务，包括：

- **服务商与模型**：支持选择主流 LLM 服务商（如 OpenAI、Anthropic、阿里云百炼、DeepSeek 等）及对应的模型版本；同时支持配置兼容 OpenAI API 协议的本地部署模型（如 Ollama、LM Studio 等），以满足数据不出本地的安全需求；
- **API Key**：填写所选服务商的访问密钥，密钥在本地存储时应加密处理；
- **接入点（Base URL）**：对于本地部署或私有化部署的服务，允许自定义接口地址；
- **连通性验证**：用户在保存配置前，可以验证当前配置是否能够正常访问目标 LLM 服务。

MVP 阶段 LLM 相关的数据生成能力尚未开放，但系统设置中的 LLM 配置入口应随产品一同发布，为后续版本做好准备。

## 11. 账号

### 11.1 背景与目的

LoomiDBX 作为桌面端软件，采用账号体系的目的是让开发商能够了解软件的实际使用情况：谁在使用、在哪里使用、使用频率如何。这些信息是产品迭代、用户支持和商业决策的基础依据。账号体系本身不用于功能权限控制（MVP 阶段），软件的核心功能对已登录用户完全开放。

### 11.2 注册

用户首次使用软件时，需要完成注册以激活使用资格。注册信息应至少包含：姓名（或昵称）、电子邮件地址、密码。邮件地址将作为唯一标识，用于后续登录和与开发商沟通（如版本更新通知、重要公告等）。注册完成后，系统应向用户的邮箱发送确认邮件以验证邮件地址的有效性。

### 11.3 登录与会话

软件启动时，若用户已完成注册，需要登录后方可使用。登录成功后，会话状态应在本地持久化，使用户在日常使用中无需反复手动登录。用户同时可以选择主动退出登录（例如在切换账号或更换设备时）。

### 11.4 数据采集说明

软件会向开发商回传必要的使用信息，包括但不限于：账号标识、软件版本、操作系统类型、软件的启动与活跃记录。**不会采集任何数据库连接信息、Schema 内容、生成器配置或生成的数据内容本身。** 隐私政策和数据采集范围应在注册时明确告知用户，并在系统设置中随时可查。
