# LoomiDBX 用户故事

## 1. 文档目的

本文档基于用户故事地图，将 LoomiDBX 的每个用户任务细化为标准用户故事格式（"作为一名 [角色]，我希望 [功能]，以便 [价值]"），并为每个故事标注 MVP 优先级与验收条件。

## 2. 用户角色定义

| 用户角色 | 简称 | 典型场景 |
|---|---|---|
| 市场/售前人员 | 售前 | 演示、方案展示、客户沟通 |
| 数据库开发/测试人员 | DBA/测试 | 测试验证、性能测试、压力压测 |
| 软件开发人员 | 开发 | 开发环境填充、联调准备 |

## 3. MVP 优先级定义

| 优先级 | 标记 | 说明 |
|---|---|---|
| P0 - 必须 | 🔴 | MVP 核心功能，必须实现 |
| P1 - 重要 | 🟡 | 重要功能，建议 MVP 包含 |
| P2 - 一般 | 🟢 | 一般功能，可延后实现 |
| P3 - 暂缓 | ⚪ | 暂不实现，留待后续版本 |

## 4. 用户故事列表

### 4.1 准备连接阶段

#### US-001: 新建数据库连接
| 项目 | 内容 |
|---|---|
| **用户故事** | 作为一名 **售前/DBA/开发**，我希望 **能够创建新的数据库连接**，以便 **将目标数据库接入 LoomiDBX 系统**。 |
| **MVP 优先级** | 🔴 P0 - 必须 |
| **验收条件** | 1. 用户可以填写连接名称、数据库类型、地址、端口、用户名、密码、其它可选参数（例如：sslmodel/connect_timeout等）<br>2. 连接信息保存后可在连接列表中查看<br>3. 连接信息支持加密存储<br>4. 必填字段未填写时提示错误 |

## 6. MVP 范围建议

基于上述用户故事，建议 MVP 版本聚焦以下范围：

### 6.1 MVP 必须包含（P0）

1. **连接管理**：新建、编辑、删除、验证数据库连接
2. **结构扫描**：浏览数据库层级结构，查看字段属性
3. **规则配置**：选择生成方式，配置规则参数，保存规则
4. **Project 管理**：创建、编辑、删除 Project，选择表，设置数量和清空策略
5. **任务执行**：启动任务，按依赖生成，查看进度，失败处理
6. **结果查看**：查看执行结果，查看历史记录，复用配置

### 6.2 MVP 建议包含（P1）

1. 重新扫描数据库结构
2. 查看结构变化影响范围
3. 预览字段配置效果
4. 复制 Project
5. 设置关联表数量倍数关系
6. 调整受影响配置
7. 批量配置字段规则

### 6.3 可延后实现（P2）

1. 导入导出配置
2. 跨表数据预览
