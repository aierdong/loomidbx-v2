# Phase 0：项目最小背景

## 阶段目标

让 Agent 用最少上下文理解 LoomiDBX 是什么、解决什么问题、核心用户是谁，以及当前版本的边界。

## 必须阅读

| 文档 | 用途 | 阅读方式 |
|---|---|---|
| [product_outline.md](../product_outline.md) | 产品定位、核心价值、功能范围 | 必读，重点读产品定位、范围、关键规则 |
| [user-store-map.md](../user-store-map.md) | 用户故事地图 | 只读概要和主干活动 |
| [user-stories.md](../user-stories.md) | 详细用户故事 | 仅在需要确认用户行为时阅读 |

## 当前阶段应形成的共识

- LoomiDBX 是桌面端数据库模拟/合成数据生成工具。
- 首发重点是连接数据库、扫描 Schema、配置字段生成规则、组织 Project、执行生成并写回数据库。
- AI 能力不是首发核心实现目标。
- UI、生成器、执行引擎、API、数据库兼容等专题会在后续阶段逐步展开。

## 本阶段产出

- 项目术语表。
- 当前版本范围与非目标。
- 后续阶段的 spec 划分基础。

## 非目标

- 不设计具体数据库表结构。
- 不实现生成器。
- 不实现 API 或 UI。
- 不处理执行引擎细节。

## Context Budget

除非当前 spec 明确要求，不要阅读生成器、执行引擎、UI DSL、数据库兼容和 AI 方案文档。
