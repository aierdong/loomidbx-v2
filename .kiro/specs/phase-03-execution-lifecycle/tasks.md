# Implementation Plan

> 任务边界约定：每个实现任务继承最近章节的边界；子任务上的显式 `_Boundary:` 和 `_Depends:` 行用于收窄该章节边界，并在存在时优先生效。

- [ ] 1. 建立生命周期包基础模型
- [x] 1.1 定义 engine 执行输入与基础快照映射
  - 建立从执行任务快照进入 lifecycle 的最小输入边界，保留任务身份、Project 引用、任务名称和表级边界。
  - 对缺少任务身份、Project 引用或表级边界的输入返回字段级预检问题，而不是创建可运行执行。
  - 完成后，有效快照可以生成不依赖 UI、Wails binding 或真实数据库连接的 engine 输入。
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_
  - _Boundary: Execution Input Mapper_
- [x] 1.2 定义生命周期安全错误和阶段表达
  - 建立生命周期错误码、阶段、字段路径和安全消息的统一表达。
  - 将输入校验、状态冲突和下游失败都约束为安全错误摘要。
  - 完成后，公开错误结果只包含安全摘要字段，不包含原始错误载荷。
  - _Requirements: 2.2, 3.3, 4.3, 5.4, 6.1, 6.2_
  - _Boundary: Safe Error Mapper_
- [x] 1.3 定义生命周期状态机基础结构
  - 定义初始化、预检中、就绪、运行中、取消中、已取消、失败和完成等 engine 内部状态。
  - 定义终态判断、流转记录和非法流转拒绝结果。
  - 完成后，新 lifecycle 实例处于可预检但尚未运行的初始状态。
  - _Requirements: 3.1, 3.3, 3.4, 3.5_
  - _Boundary: Lifecycle State Machine_

- [ ] 2. 实现预检与状态流转核心语义
- [x] 2.1 实现预检结果汇总和启动前阻断规则
  - 汇总基础输入校验、生命周期状态前置条件和下游预检结果。
  - 区分阻断错误和非阻断警告，并根据阻断错误计算通过状态。
  - 完成后，存在阻断错误的执行不会进入运行状态，预检结果可直接展示安全摘要。
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5_
  - _Boundary: Precheck Aggregator_
- [x] 2.2 实现合法状态流转和终态互斥规则
  - 实现初始化到预检、预检到就绪或失败、就绪到运行、运行到取消/失败/完成的允许流转。
  - 拒绝终态后的再次启动、完成、失败或取消请求。
  - 完成后，取消、失败和完成只能形成一个互斥终态，并保留可测试流转记录。
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_
  - _Boundary: Lifecycle State Machine_
- [x] 2.3 实现取消意图和下游可观察控制边界
  - 为运行中执行提供取消请求入口和可观察取消标记。
  - 使后续下游步骤能够读取取消状态，但不实现 goroutine 强制中断或数据库取消。
  - 完成后，运行中的取消请求会标记取消意图并推进到取消终态路径。
  - _Requirements: 4.2, 4.5_
  - _Boundary: Control Token_

- [ ] 3. 接入生命周期协调器和下游接缝
- [ ] 3.1 实现生命周期协调入口
  - 串联输入映射、预检、启动、当前边界内下游阶段、失败、取消和完成语义。
  - 启动时记录启动时间和执行上下文，成功结束时记录完成时间。
  - 完成后，调用方可以通过一个 engine 入口获得运行、失败、取消或完成后的最终快照。
  - _Requirements: 2.2, 3.2, 4.1, 4.3, 4.4, 4.5_
  - _Boundary: Lifecycle Coordinator_
- [ ] 3.2 定义计划、生成和结果汇总阶段的最小接缝
  - 为计划、生成和结果汇总阶段建立可替换 hook，使后续规格能够接入各自实现。
  - 当前接缝只返回阶段成功、阶段失败或安全错误，不实现依赖图、行数规划、生成器注册表、批量循环、result aggregator 或真实写入。
  - 完成后，no-op 或 fake 接缝可被协调器调用并驱动生命周期状态变化。
  - _Requirements: 1.5, 2.4, 5.1, 5.2, 5.3, 5.5_
  - _Boundary: Downstream Ports_
- [ ] 3.3 将下游失败映射为生命周期失败终态
  - 当计划、生成或结果汇总任一接缝返回失败时，将失败转换为任务级生命周期失败语义。
  - 失败结果只保留阶段、错误码、字段路径和安全消息。
  - 完成后，下游原始错误不会透出，lifecycle 会进入失败终态并提供最终快照。
  - _Requirements: 4.3, 5.4, 6.1, 6.2_
  - _Boundary: Lifecycle Coordinator, Safe Error Mapper_
- [ ] 3.4 生成最终生命周期快照和历史映射边界
  - 输出状态、启动时间、结束时间、取消意图、失败摘要和流转记录。
  - 只在最终快照边界表达到 Phase 2 历史状态的映射，不修改 Phase 2 枚举。
  - 完成后，取消、失败或完成后的快照结构一致，可供后续历史记录、API 或 UI 规格消费。
  - _Requirements: 3.5, 4.1, 4.4, 4.5_
  - _Boundary: Lifecycle Snapshot_

- [ ] 4. 覆盖核心行为测试
- [ ] 4.1 覆盖输入映射和预检结果测试
  - 验证有效快照通过预检并生成 engine 输入。
  - 验证缺失任务身份、Project 引用或表边界时返回字段级阻断错误。
  - 验证预检警告不会阻止启动，阻断错误一定阻止启动。
  - 完成后，输入与预检测试覆盖基础校验、汇总结果和未运行状态边界。
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 2.4_
  - _Boundary: Execution Input Mapper, Precheck Aggregator_
- [ ] 4.2 覆盖状态机、取消、失败和完成测试
  - 验证初始化、合法流转、非法流转和终态拒绝规则。
  - 验证启动时间、取消意图、失败摘要、完成时间和流转记录。
  - 完成后，状态机测试证明取消、失败和完成是互斥终态。
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 4.1, 4.2, 4.3, 4.4, 4.5_
  - _Boundary: Lifecycle State Machine, Control Token, Lifecycle Snapshot_
- [ ] 4.3 覆盖安全错误过滤测试
  - 验证生命周期、预检和下游接缝错误只暴露错误码、阶段、字段路径和安全消息。
  - 使用包含密码、用户 SQL、连接详情和生成数据内容的输入确认公开消息已安全化。
  - 完成后，测试输出中不会出现敏感原文。
  - _Requirements: 6.1, 6.2_
  - _Boundary: Safe Error Mapper_

- [ ] 5. 覆盖接缝与边界验证
- [ ] 5.1 覆盖下游接缝替换和失败传播测试
  - 使用 fake/no-op 计划、生成和结果汇总接缝验证成功路径按顺序完成。
  - 分别模拟计划、生成和结果汇总失败，验证统一失败终态和安全摘要。
  - 完成后，替换不同接缝实现不会改变生命周期状态规则。
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5, 6.5_
  - _Boundary: Downstream Ports, Lifecycle Coordinator_
- [ ] 5.2 覆盖 engine 包依赖边界测试
  - 验证 lifecycle 包不依赖 Wails、Vue、前端 API 或真实数据库驱动。
  - 验证本规格没有修改 Phase 2 执行历史状态枚举来表达 lifecycle 内部状态。
  - 完成后，边界测试可阻止状态规则流入 binding/UI 或数据库写入层。
  - _Requirements: 1.4, 3.1, 3.5, 6.4_
  - _Boundary: Boundary Tests_
- [ ] 5.3 覆盖未来能力未提前实现的边界测试
  - 验证测试和实现中不存在依赖图算法、行数规划、生成器注册表、批量生成循环、result aggregator 或真实写入行为。
  - 验证下游接缝只表达阶段结果和安全错误，而不承载未来规格的业务算法。
  - 完成后，本规格可通过测试证明只实现生命周期边界，不吸收后续 Phase 3 能力。
  - _Requirements: 2.5, 5.1, 5.2, 5.3, 6.5_
  - _Boundary: Boundary Tests, Seam Tests_
