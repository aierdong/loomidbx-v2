# Implementation Plan

- [x] 1. 建立本地存储基础配置与布局
- [x] 1.1 定义本地存储配置、路径布局和诊断视图
  - 将配置系统提供的数据目录作为唯一输入，不新增第二套路由或环境解析。
  - 明确主业务数据库、迁移、临时和备份预留目录的相对布局。
  - 完成后，调用方可以从一个诊断视图看到数据目录、数据库文件和迁移状态字段。
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 6.2_
- [x] 1.2 实现布局校验和隔离目录行为
  - 校验目录可创建、可写和路径格式错误。
  - 支持开发/测试隔离数据目录，避免触碰真实用户目录。
  - 完成后，无效目录返回稳定错误，测试模式初始化只写入临时或隔离路径。
  - _Requirements: 2.3, 2.4, 6.3_

- [x] 2. 建立 SQLite 初始化和迁移基础设施
- [x] 2.1 实现本地存储初始化入口
  - 创建布局目录，打开主业务数据库，并保持初始化入口不依赖前端 UI。
  - 初始化只建立基础设施，不创建完整连接、Schema、Project 或执行历史业务表。
  - 完成后，在空数据目录中运行初始化会得到 ready 诊断和可打开的数据文件。
  - _Requirements: 3.1, 6.1, 6.2_
- [x] 2.2 实现迁移注册、排序和记录机制
  - 提供递增编号迁移命名规则和基础迁移记录结构。
  - 只执行尚未成功记录的迁移，并记录成功版本。
  - 完成后，重复初始化不会重复执行已成功迁移。
  - _Requirements: 3.2, 3.4_
- [x] 2.3 实现迁移失败处理
  - 迁移执行失败时返回稳定错误码和失败原因。
  - 失败迁移不得写入成功记录，修复后可重试。
  - 完成后，故意失败的迁移测试能证明状态未被标记为成功。
  - _Requirements: 3.3, 6.3_

- [x] 3. 建立 Repository 边界与测试替身
- [x] 3.1 定义 Repository 接口和工作单元边界
  - 分离服务层接口、SQLite 实现落位和底层连接管理。
  - 提供最小工作单元约定，避免服务层直接操作数据库连接。
  - 完成后，服务层可以依赖 repository 契约读取存储诊断而不接触文件路径细节。
  - _Requirements: 4.1, 4.2_
- [x] 3.2 (P) 建立 Repository 错误语义和未实现能力响应
  - 定义未实现、未迁移和存储不可用等稳定错误。
  - 未实现业务表或后续仓储不得静默创建不受控结构。
  - 完成后，调用未完成业务仓储会返回明确错误而不是空成功。
  - _Requirements: 4.4, 6.3_
  - _Boundary: RepositoryContracts_
- [x] 3.3 (P) 建立 mock 或内存 Repository 替身策略
  - 提供测试替身落位和使用规则。
  - 替身遵守真实接口的错误语义，不声明完整业务仓储已实现。
  - 完成后，服务层单元测试可以不依赖真实 SQLite 文件验证存储相关行为。
  - _Requirements: 4.3, 6.4_
  - _Boundary: MockRepositories_

- [x] 4. 建立敏感信息隔离边界
- [x] 4.1 定义 SecretRef 和 SecretStore 不可用实现
  - 只允许持久化凭据引用、配置状态或脱敏元数据。
  - 默认 secret store 返回不可用状态，不降级为明文保存。
  - 完成后，连接凭据相关调用可以表达“未配置/不可用”而不会写入明文。
  - _Requirements: 1.4, 5.2, 5.3_
- [x] 4.2 实现敏感错误和诊断脱敏规则
  - 错误、日志候选文本和诊断视图不得包含密码、token、用户 SQL 或生成数据内容。
  - 存储策略明确本地产品数据不上传远端。
  - 完成后，敏感输入测试只能在错误中看到字段名、错误码或脱敏说明。
  - _Requirements: 5.1, 5.4, 6.3_

- [x] 5. 集成验证与文档边界更新
- [x] 5.1 更新 storage 和 repository 边界说明
  - 将占位 README 更新为已实现策略、非目标、数据分类和后续规格接入方式。
  - 明确完整业务表、完整 Repository、安全存储和 UI 均由后续规格完成。
  - 完成后，开发者查看目录说明即可判断新存储代码应放在哪里、哪些能力尚未完成。
  - _Requirements: 1.1, 1.2, 1.3, 4.1, 5.1_
- [x] 5.2 增加本地存储基础设施测试
  - 覆盖布局解析、初始化、迁移幂等、迁移失败、Repository 替身和敏感信息脱敏。
  - 测试必须使用隔离目录，不能读写真实用户目录。
  - 完成后，项目测试可以验证本地存储底座，无需完整业务模型存在。
  - _Requirements: 2.3, 2.4, 3.1, 3.2, 3.3, 4.3, 5.4, 6.4_
- [x] 5.3 执行端到端初始化回归
  - 在测试隔离目录中完成配置输出、存储初始化、迁移执行和 repository 诊断读取的闭环。
  - 验证初始化失败时返回稳定错误码并保持敏感信息脱敏。
  - 完成后，后续规格可以把该回归作为接入本地存储的基础验收。
  - _Requirements: 2.1, 3.1, 3.2, 4.1, 5.1, 6.1, 6.2, 6.3_
